# 按计划编码实施

## 概述

严格按照编码计划逐步执行代码实现。自动创建 progress.md 跟踪执行进度，通过 Hook 提醒机制确保不遗漏关键步骤。

## 前置步骤（必须执行）

1. **读取编码计划**：搜索 `.ai-configs/plan/` 下最新的编码计划
2. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
3. **创建 progress.md**：在 `.ai-configs/progress/YYYY/MM/` 下创建执行进度文件
4. **理解约束条件**：确认用户的硬性要求和限制

## 执行原则

- **严格遵循计划**：不偏离、不添加计划外功能
- **遵守约束条件**：用户明确的硬性要求
- **步步为营**：完成一步再进行下一步
- **透明沟通**：关键节点及时确认

## Hook 提醒机制

### 开始前检查
- [ ] 是否已读取编码计划？
- [ ] 是否已创建 progress.md 文件？
- [ ] 是否理解所有约束条件？

### 执行中提醒（每完成一步）
- [ ] 是否更新了 progress.md 的执行日志？
- [ ] 是否标记了 Todo 状态为 completed？
- [ ] 是否记录了遇到的错误（如有）？

### 完成后验证
- [ ] 所有 Todo 是否标记为 completed？
- [ ] 是否运行了构建测试（npm run build）？
- [ ] progress.md 是否已更新完整？

### 错误发生时
1. 立即停止当前操作
2. 在 progress.md 中记录错误（时间、现象、根因、尝试方案）
3. 分析是否需要调整计划
4. 记录经验教训

## 输出契约

**产出**：
- 修改/创建的代码文件
- `.ai-configs/progress/YYYY/MM/[功能名称]-执行进度.md`

progress.md 包含：执行日志、错误记录、测试结果、待办事项、进度统计。

## 核心原则

- **计划优先**：严格按计划执行，如需调整先与用户确认
- **进度透明**：实时更新 progress.md
- **错误记录**：避免重复踩坑
- **质量保证**：完成后必须验证构建

## 工作流衔接

| 上游 | 下游 |
|------|------|
| `/create-plan` 编码计划 | → `/review` 代码审查 |
| `/update-plan` 更新后的计划 | → 手动测试和提交 |

完成后建议：
1. 运行 `npm run build` 验证构建
2. 使用 `/review` 审查代码变更
3. 提交代码并创建 PR
