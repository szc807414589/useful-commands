# PRD 文档生成

## 概述

基于需求分析生成产品需求文档。智能复用已有分析报告，通过引用而非复制避免信息冗余。

## 前置步骤（必须执行）

1. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
2. **检查已有分析报告**：搜索 `.ai-configs/analysis/` 下相关文档
   - 如果存在分析报告 → 引用而非复制（节省 50-70% 时间）
   - 如果无分析报告 → 轻量级需求理解

## 核心流程

### 智能复用机制

- 有分析报告 → 引用报告中的探索发现和技术细节
- 无分析报告 → 进行轻量级需求理解，建议后续补充分析

### PRD 生成

1. 功能概要（1-2 行，最多 50 字）
2. 背景与目标
3. 用户场景
4. 功能需求（详细描述）
5. 技术方案（记录决策，引用探索发现）
6. 验收标准（明确可测试）
7. 风险评估（仅记录产品层面新增风险）
8. 里程碑与优先级

## 输出契约

**核心原则**：
- 功能概要：1-2 行（最多 50 字）
- 技术方案：记录决策，引用探索发现
- 风险评估：仅记录产品层面新增风险
- 不复制分析报告的详细内容

**产出文件**：`.ai-configs/prd/YYYY/MM/[功能名称].md`

## 核心原则

- **引用 > 复制**：引用已有分析报告，不重复探索
- **决策驱动**：记录技术决策和原因，不只是描述
- **验收明确**：每个功能点都有可测试的验收标准
- **精简高效**：避免冗余描述，信息单一来源

| 用户直接描述需求 | → `/update-prd` 后续更新 |
