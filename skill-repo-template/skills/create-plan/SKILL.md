# 编码计划生成

## 概述

生成详细的 Phase 复选框格式编码实施计划。智能复用已有分析报告和 PRD，通过三层引用结构避免重复探索。

## 前置步骤（必须执行）

1. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
2. **检查已有文档**（智能复用）：
   - 有分析报告 → 直接复用探索结果（节省 60-80% 时间）
   - 有 PRD（基于分析）→ 读取 PRD + 引用分析报告
   - 有 PRD（无分析）→ 读取 PRD + 补充探索
   - 无任何文档 → 强烈建议先运行 `/req-analyze`

## 核心流程

1. **功能概要**：3-5 行（最多 150 字）
2. **参考实现索引**：纯索引表（文件路径 + 行号），不复制代码
3. **实施步骤**：Phase 复选框格式
4. **风险评估**：仅记录实施层面风险
5. **验收标准**：可测试的检查点

### Phase 复选框格式

```markdown
Phase 1: 基础组件搭建
- [ ] 步骤 1.1: 创建导出按钮组件
  - 涉及文件：`src/components/ExportButton.jsx`
  - 具体任务：
    - [ ] 创建组件文件
    - [ ] 实现点击事件
    - [ ] 添加样式
  - 验收：按钮显示正常，点击触发事件

Phase 1 进度: 0/4 (0%)
总进度: 0/8 (0%)
```

### 三层引用结构

```
编码计划（如何实施）
  ↓ 引用
PRD 文档（技术决策和验收标准）
  ↓ 引用
需求分析报告（探索发现和技术细节）
```

## 输出契约

**核心原则**：
- 功能概要：3-5 行（最多 150 字）
- 参考实现：纯索引表（文件路径 + 行号）
- 实施步骤：Phase 复选框格式
- 不复制代码片段
- 计划阶段不读取详细代码

**产出文件**：`.ai-configs/plan/YYYY/MM/[功能名称]-编码计划.md`

## 核心原则

- **引用 > 复制**：引用已有文档，不重复探索
- **Phase 结构**：清晰的阶段划分和进度追踪
- **可执行性**：每个步骤足够具体，可直接执行
- **索引化**：参考实现只记录路径和行号
