# 脑暴与方向探索

## 概述

当需求方向不清晰、需要发散思维比较多个方案时使用。专注于发散和收敛思考，不做 PRD、编码计划或具体代码实现。

## 前置步骤（必须执行）

1. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
   - 如果文件存在，获取业务领域、技术栈、设计规范、已有约束
   - 如果文件不存在，根据当前输入直接进入对话

2. **检查已有文档**（复用已有结论）
   ```bash
   glob ".ai-configs/brainstorm/**/*关键词*.md"
   glob ".ai-configs/analysis/**/*关键词*.md"
   glob ".ai-configs/design/**/*关键词*.md"
   ```
   如果找到相关文档，提取已有结论，避免重复讨论已决定的内容。

## 核心流程

**角色**：产品策略与方案脑暴助手

1. **问题 framing**：当前现状、改善指标、核心用户、卡点原因
2. **场景拆解**：主场景、次场景、边界场景、失败场景、未来扩展
3. **方案发散**：至少 3 类不同方向（非同一方案的微调）
4. **约束与未知项识别**：产品边界、依赖、限制、风险
5. **推荐方向**：为什么选、为什么不选其他、适合当前还是长期

## 输出契约（6 部分）

1. 问题定义（背景、目标、触发原因）
2. 核心场景（用户、场景、频率、价值）
3. 方案选项（至少 3 个方向，含优缺点）
4. 风险与未知项
5. 推荐方向（主推方案及理由）
6. 下一步建议

**产出文件**：`.ai-configs/brainstorm/YYYY/MM/[主题]-脑暴.md`

## 核心原则

- **发散优先**：先打开思路，再收敛方向
- **方向级差异**：方案之间是不同方向，不是同一方案的变体
- **决策有据**：推荐方向必须有明确理由
- **不做实施**：只负责方向探索，不做 PRD 或编码计划

## 工作流衔接

| 方向 | 推荐 |
|------|------|
| 方向已定，需深度分析 | → `/req-analyze` |
| 方向已定，需设计文档 | → `/design` |
| 方向已定，需产品需求文档 | → `/prd` |
