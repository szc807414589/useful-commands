# 设计方案生成

## 概述

输出可交付、可实施的设计方案文档，包含 8 个标准章节。适用于页面改版、功能重构、复杂交互设计等场景。

## 前置步骤（必须执行）

1. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
   - 确认业务领域、技术栈、设计规范、已有约束
2. **读取 package.json**：确认 UI 组件库版本、样式方案
3. **检查已有文档**（复用已有结论）
   ```bash
   glob ".ai-configs/brainstorm/**/*关键词*.md"
   glob ".ai-configs/analysis/**/*关键词*.md"
   glob ".ai-configs/design/**/*关键词*.md"
   ```

## 核心流程

**角色**：产品设计与交互设计助手

1. **收敛设计目标**：明确设计边界和成功标准
2. **定义信息架构**：页面结构、导航层级、内容组织
3. **定义关键交互**：主流程、次流程、回退、异常
4. **定义界面规则**：容器、布局、组件选型
5. **定义视觉与文案约束**：色彩、字体、间距、文案规范
6. **定义边界状态**：加载、空、错误、权限、极端数据

## 输出契约（8 部分）

1. 设计目标与约束
2. 用户角色与关键场景
3. 信息架构/页面结构
4. 关键交互流程
5. 界面布局与组件规则
6. 视觉与文案规范
7. 状态设计（加载、空、错误、权限、边界）
8. 设计交接清单（给开发和测试）

**产出文件**：`.ai-configs/design/YYYY/MM/[功能名称]-设计方案.md`

## 核心原则

- **可实施性**：设计方案必须可直接落地为代码
- **完整覆盖**：8 个章节缺一不可
- **边界优先**：特别关注异常状态和边界情况
- **组件化思维**：基于现有 UI 库进行组件选型
