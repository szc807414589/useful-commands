# PRD 更新

## 概述

根据需求变更更新现有 PRD 文档。支持三种更新模式，确保更新后文档各部分保持逻辑一致。

## 前置步骤（必须执行）

1. **读取现有 PRD**：搜索 `.ai-configs/prd/` 下相关 PRD 文件
   - 如果找不到 → 提示用户先使用 `/prd` 创建
2. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
3. **理解变更需求**：明确哪些部分需要更新、为什么更新

## 核心流程

1. **读取现有 PRD** → 理解当前文档结构和内容
2. **理解更新需求** → 明确变更范围和原因
3. **分析影响** → 评估变更对其他部分的连锁影响
4. **生成更新方案** → 列出具体修改点
5. **用户确认** → 确认更新方案
6. **执行更新** → 修改 PRD 文件
7. **记录更新日志** → 在 PRD 末尾追加变更记录

## 更新模式

| 模式 | 适用场景 | 特点 |
|------|---------|------|
| **局部更新** | 细节补充、验收调整、文案优化 | 小范围修改，不影响整体结构 |
| **重大更新** | 产品方向、范围或方案明显变化 | 评估是否升版本，全面检查一致性 |
| **增量更新** | 增加功能模块、验收点或风险措施 | 在现有结构上追加内容 |

## 输出契约

- 更新现有 PRD 文件（保持文件路径不变）
- 在 PRD 末尾追加更新日志（日期、变更内容、原因）

## 核心原则

- **一致性**：更新后各部分保持逻辑一致
- **可追溯**：记录每次变更的原因和内容
- **最小改动**：只修改必要的部分，不做无关调整
- **影响评估**：重大变更后提醒同步运行 `/update-plan`

## 工作流衔接

| 上游 | 下游 |
|------|------|
| 需求变更 | → `/update-plan` 同步更新编码计划 |
| 用户反馈 | → `/create-plan` 重新生成计划（重大变更时） |
