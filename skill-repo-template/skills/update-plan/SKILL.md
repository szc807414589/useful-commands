# 编码计划更新

## 概述

根据需求变更或新发现更新现有编码计划。支持三种更新模式，确保更新后步骤依赖关系正确、逻辑一致。

## 前置步骤（必须执行）

1. **读取现有编码计划**：搜索 `.ai-configs/plan/` 下相关计划文件
   - 如果找不到 → 提示用户先使用 `/create-plan` 创建
2. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
3. **理解变更需求**：明确哪些部分需要更新、为什么更新
4. **检查执行进度**：搜索 `.ai-configs/progress/` 查看已执行的步骤

## 核心流程

1. **读取现有计划** → 理解当前计划结构和进度
2. **理解更新需求** → 明确变更范围和原因
3. **分析影响** → 评估变更对步骤依赖的连锁影响
4. **探索代码库**（如需）→ 补充新的技术发现
5. **生成更新方案** → 列出具体修改点
6. **用户确认** → 确认更新方案
7. **执行更新** → 修改计划文件
8. **记录更新日志** → 在计划末尾追加变更记录

## 更新模式

| 模式 | 适用场景 | 特点 |
|------|---------|------|
| **局部更新** | 微调步骤、修正遗漏、更新路径 | 小范围修改，不影响整体结构 |
| **重大更新** | 技术方案、目标或约束大幅变化 | 评估是否升版本，全面检查依赖关系 |
| **增量更新** | 增加步骤、功能或风险控制 | 在现有结构上追加内容 |

## 输出契约

- 更新现有计划文件（保持文件路径不变）
- 在计划末尾追加更新日志（日期、变更内容、原因）

## 核心原则

- **依赖正确**：确保更新后步骤依赖关系正确
- **一致性**：更新后各部分逻辑一致
- **进度感知**：如已开始编码，建议重新审查已完成部分
- **可追溯**：记录每次变更的原因和内容

## 工作流衔接

| 上游 | 下游 |
|------|------|
| `/update-prd` PRD 变更后 | → `/code-by-plan` 按更新后计划继续 |
| 执行中发现问题 | → 继续编码实施 |
