# 需求分析

## 概述

对新需求进行深度分析，包括代码库探索和可行性评估。产出结构化分析报告，供后续 PRD 和编码计划引用，避免重复探索。

## 前置步骤（必须执行）

1. **读取项目上下文**：使用 Read 工具读取 `.claude/project-context.md`
2. **读取 package.json**：确认当前项目的依赖版本
3. **检查已有文档**：搜索 `.ai-configs/analysis/` 下是否已有相关分析

## 核心流程

1. **需求澄清**（15%）：明确需求范围、边界、验收标准
2. **代码库探索**（40%）：核心阶段，实时记录发现
   - 搜索相关模块、组件、函数
   - 找到参考实现（文件路径 + 行号）
   - 理解现有架构和数据流
3. **影响面分析**（25%）：直接影响 + 间接影响 + 跨模块影响
4. **可行性评估**（15%）：技术方案、工作量估算、风险识别
5. **生成分析报告**（5%）：使用模板输出结构化报告

### 实时记录发现

探索过程中带时间戳记录：
```markdown
⏱️ 10:30 找到相关模块：`src/monitor/alert/`
⏱️ 10:35 参考实现：`AlertHistory.jsx:234-289`
⏱️ 10:40 关键依赖：`xlsx@0.18.5`
```

## 输出契约

产出结构化分析报告，包含：需求理解、代码探索发现、影响面分析、可行性评估、风险清单、建议方案。

**产出文件**：`.ai-configs/analysis/YYYY/MM/[需求名称]-需求分析.md`

## 核心原则

- **探索 > 假设**：通过代码探索找到真相，不基于假设下结论
- **实时记录**：探索过程中带时间戳记录发现
- **引用而非复制**：报告中引用文件路径和行号，不复制代码片段
- **全面覆盖**：搜索所有相关代码，不遗漏影响面

## 工作流衔接

| 上游 | 下游 |
|------|------|
| `/brainstorm` 脑暴后深化 | → `/prd` 生成产品需求文档 |
| 用户直接提需求 | → `/create-plan` 生成编码计划 |

完成后根据复杂度推荐：
- 复杂功能 → `/prd`
- 中等功能 → `/create-plan`
- 简单修复 → `/code-by-plan`
